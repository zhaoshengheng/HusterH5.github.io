# 地标图片资源说明

本目录存放 14 处校园地标的 **未来重构图**（AI 生成），每张图按身份区分。
真实地标照片（"现在"图）存放在根目录 `图片/` 文件夹，不在此处。

## 四身份编号与文件名

每个地标文件夹内有 4 张图，按身份编号：

| 编号 | 身份 key | 文件名（标准） |
|------|---------|---------------|
| 1 | highschool | `1高中生.png` |
| 2 | student | `2在校生.png` |
| 3 | alumni | `3毕业校友.png` |
| 4 | public | `4社会大众.png` |

**两个特殊文件名**（历史遗留）：
- 南大门（id=1）校友：`3已毕业校友.png`（非"毕业校友"）
- 青年园（id=4）公众：`4社会人士.png`（非"社会大众"）

## ⚠️ 文件夹编号与地标 id 不一致

11~14 的文件夹编号与地标 id 存在错位（历史遗留），代码中通过 `futureFolders` 映射表对应：

| 地标 id | 文件夹编号 | 文件夹名 |
|---------|-----------|---------|
| 11 | 12 | `12碧珠长廊` |
| 12 | 11 | `11图书馆` |
| 13 | 14 | `14同济医学院大门` |
| 14 | 13 | `13水塔` |

对应逻辑见 `script.js` 中的 `getFutureSrc()` 函数。

## 缺图回退

若未来图缺失，H5 会回退到 `assets/landmarks/XX_future.jpg`（旧版统一图），再缺失则显示占位框，不影响交互。
